# نوا · Nava — سرویس استریم موسیقی (فاز اول)

پروژه درس **برنامه‌سازی وب**. نوا یک سرویس استریم موسیقی شبیه Spotify است با چهار
نقش کاربری (شنونده، هنرمند، پشتیبان، مدیر). این مخزن **فاز اول (فرانت‌اند)** را
پیاده‌سازی می‌کند: تمام رابط کاربری، واکنش‌گرا (دسکتاپ/تبلت/موبایل)، با داده‌های
ساختگی که در **LocalStorage** نگه‌داری می‌شوند. معماری طوری چیده شده که **فاز دوم
(بک‌اند جنگو)** صرفاً با جایگزینی لایهٔ مخزن (Repository) انجام شود، نه بازنویسی.

> Nava is a Spotify-like music streaming service built for the Web Programming
> course. This repository delivers **Phase 1 (frontend only)** with mock data in
> LocalStorage, designed so **Phase 2 (Django backend)** is a swap, not a rewrite.

---

## اجرا · Running

```bash
pnpm install
pnpm dev          # http://localhost:3000  (Turbopack)
```

اسکریپت‌های دیگر · Other scripts:

```bash
pnpm build        # بیلد تولید (webpack — برای سرویس‌ورکر Serwist)
pnpm start        # اجرای بیلد تولید
pnpm lint         # ESLint
pnpm typecheck    # tsc --noEmit
pnpm test         # Vitest (۵۰ تست)
```

داده‌های نمونه در اولین بارگذاری به‌طور خودکار در LocalStorage ریخته می‌شوند
(`lib/db/seed.ts`). برای ریست، LocalStorage مرورگر را پاک کنید.

## معماری · Architecture

سه لایهٔ تمیز تا تعویض منبع داده فقط یک پوشه را لمس کند:

```
lib/db/storage.ts   LocalStorage تایپ‌دار + کلیدهای فضای‌نام‌دار
lib/db/seed.ts      داده‌های نمونهٔ منسجم و idempotent
lib/api/*.ts        مخزن‌ها — توابعی که Promise برمی‌گردانند
                    فاز ۱: خواندن/نوشتن LocalStorage
                    فاز ۲: همان امضاها، بدنه = ky → جنگو
lib/queries/*.ts    هوک‌های TanStack Query — بین دو فاز بدون تغییر
```

- **State:** Zustand برای پخش‌کننده/احراز هویت/UI (`store/*`)، TanStack Query
  برای دادهٔ «سروری».
- **محدودیت اشتراک‌ها** (`lib/subscriptions.ts`): جدول ۱ به‌صورت پیکربندی کد شده
  (تنها منبع حقیقت برای هر گیت). قیمت‌ها در DB ذخیره می‌شوند تا مدیر بدون تغییر کد
  آن‌ها را ویرایش کند.
- **گیت نقش‌ها** (`lib/auth/permissions.ts`): «دسترسی بالاتر از سطح ممنوع».
- **دو زبانه** (`lib/i18n/`): فارسی/انگلیسی واقعی، RTL/LTR، پیش‌فرض فارسی.

### پشتهٔ فناوری · Tech stack

Next.js 16 (App Router) · React 19 · TypeScript · Tailwind v4 · shadcn (base-ui,
RTL) · TanStack Query · Zustand · react-hook-form + zod · motion · next-themes ·
sonner · recharts · jalaali-js · Serwist (PWA) · Vitest + Testing Library.

### طراحی · Design

«Aurora Glass» — گلسمورفیسم بنفش تیره: سطوح شیشه‌ای نیمه‌شفاف، درخشش نئونی ملایم،
گوشه‌های گرد بزرگ، نوار پخش شناور و ناوبری موبایل شیشه‌ای.

## امکانات پیاده‌سازی‌شده · Features

- **ورود/ثبت‌نام:** ورود مشترک چهار نقش با هدایت بر اساس نقش، فراموشی رمز،
  ثبت‌نام شنونده (تاریخ تولد جلالی، جنسیت، مودال حریم خصوصی الزامی)، درخواست
  هنرمند با وضعیت «در انتظار تأیید».
- **خانه:** نام و آواتار، اخیراً شنیده‌شده، تازه‌ترین آلبوم/تک‌آهنگ، بخش دسترسی
  زودهنگام طلایی.
- **پخش‌کننده موسیقی (۸۰۰ امتیاز):** نوار پخش دسکتاپ، مینی‌پلیر موبایل و تمام‌صفحه،
  نوار پیشرفت قابل جابه‌جایی، بعدی/قبلی، صدا، صف پخش (جابه‌جایی/حذف)، تکرار سه‌حالته،
  پخش تصادفی، متن آهنگ، لینک هنرمند/آلبوم، آمار فقط برای طلایی، گیت ۶۰ استریم/روز.
- **مرور:** جستجو بر اساس اثر یا هنرمند + مرتب‌سازی (شنونده/تاریخ)، شبکهٔ آلبوم و تک‌آهنگ.
- **پروفایل‌ها:** پروفایل هنرمند (بیو، آثار، نشان تأیید، آمار طلایی، دنبال کردن)،
  پروفایل کاربر (هندل سیستمی، نشان اشتراک، دنبال‌کننده/دنبال‌شونده، استریم روزانه،
  ویرایش پروفایل با آواتار گیت‌شده برای پایه).
- **پلی‌لیست‌ها:** ساخت/تغییر نام/حذف، محدودیت ۶/۱۰۰/∞، حالت خالی، افزودن آهنگ.
- **تنظیمات:** اعلان، صدای سامانه (همگام با پخش‌کننده)، زبان واقعی، پوسته، حذف حساب،
  مشاهدهٔ اشتراک و ارتقا (پرداخت = فاز ۲).
- **اعلانات:** کارت‌های مخصوص نقش، خوانده/نخوانده با نقطهٔ آبی، خواندن/حذف،
  خواندن همه، حالت خالی، لینک مستقیم به اثر.
- **استودیو هنرمند:** آپلود (MP3/WAV/FLAC)، متن، کاور، نوع، ژانر/سال/همکاران،
  آمار هر اثر، ویرایش/حذف.
- **داشبورد:** تیکت‌ها + چت‌باکس، احراز هویت + تأیید/رد با علت (+اعلان به هنرمند)،
  حسابرسی + تسویه (مدیر)، کنترل قیمت اشتراک + نمودار دایره‌ای + ویجت درآمد (مدیر).
- **PWA:** سرویس‌ورکر Serwist + manifest، قابل نصب.

## فاز دوم (پیاده‌سازی‌نشده) · Phase 2 outlook

مدل‌های جنگو (User/Song/Album/Playlist و …)، CRUD رست‌فول با DRF، اعمال محدودیت
اشتراک و دسترسی نقش‌ها در سمت سرور، آپلود فایل، درگاه پرداخت، گزارش‌گیری تجمیعی.
بدنهٔ `lib/api/*` از LocalStorage به `ky` تغییر می‌کند؛ هوک‌های Query و کامپوننت‌ها
دست‌نخورده می‌مانند. پروکسی `/api` در `next.config.mjs` به `localhost:8321` آماده است.
